## Чего нет как отдельного `.py` в репо (и зачем)

| Модуль | Почему |
|--------|--------|
| `main.py` (~15 000 строк) | Общий backend LBL Studio + Cloud; в репо только **выдержки** в `backend/snippets/` |
| `database.py` | Модели всей платформы |
| `utils.py` (полный) | Почта студии и Cloud; в репо — **фрагмент** для листинга А.15 |

Для защиты это нормально: в записке описан **модульный** проект Cloud, а общий `main.py` монтирует `lbl_drive` и `cloud_billing`.

## Как вставлять в Word

1. Открыть файл из таблицы в GitHub или в IDE.
2. Выделить нужные строки (как в записке — часто 20–40 строк).
3. В Word: **Вставка → Объект → Текст из файла** или вставка с моноширинным шрифтом (Consolas 9–10 pt).
4. Подпись: «Листинг А.N — …»

Скриншоты интерфейса — **Приложение Б** (сделать с https://cloud.lbl3d.info или `?demo=1` локально).
