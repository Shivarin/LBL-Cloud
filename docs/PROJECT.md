# LBL Cloud — описание проекта (портфолио)

## Задача

Нужен **отдельный облачный продукт** для пользователей LBL: хранить файлы, делиться внутри команды, платить за расширенную квоту — без ощущения, что это «ещё одна страница сайта 3D-студии».

## Что сделано

1. **Публичный лендинг** — позиционирование, тарифы, CTA, адаптив, анимации (`frontend/landing/`).
2. **Веб-приложение `/app/`** — интерфейс близкий к Google Drive: боковое меню, список/сетка, контекстные действия, вкладка «Аккаунт» с безопасностью (`frontend/app/`).
3. **API диска** — папки, browse по разделам (диск / недавние / избранное / корзина), upload simple + chunked, превью, soft delete (`backend/lbl_drive.py`).
4. **Биллинг** — тарифы, checkout ЮKassa, webhook активации подписки (`backend/cloud_billing.py`).
5. **Безопасность** — 2FA при входе, включение/отключение 2FA в профиле, смена пароля, письма с брендом Cloud.
6. **Продакшен** — nginx для `cloud.lbl3d.info`, скрипты деплоя и проверки, лимит тела 1 ГБ для загрузок.

## Технические решения

### Почему vanilla JS, а не React

Диск уже жил в монорепозитории LBL как отдельный модуль; для скорости выката и единого стиля с остальными страницами оставили **без сборщика**: один `lbl-drive.js`, CSS-модули по зонам (layout, profile, uploads). Меньше зависимостей на сервере, проще отдавать статику через nginx.

### Chunked upload

Большие файлы идут через `upload/init` → `upload/chunk` → `upload/complete`, чтобы не упираться в таймауты proxy и держать контроль квоты на каждом чанке.

### Отдельный бренд писем

`resolve_email_brand()` смотрит `app_context` и `Origin`: с `cloud.lbl3d.info` пользователь получает письма «LBL Cloud ID», не «LBL Studio».

### UI аккаунта

Блок «Безопасность и вход» вынесен в отдельные классы (`ld-account-card`), чтобы старые стили промо не скрывали 2FA и форму пароля на проде.

## Сложности

| Проблема | Решение |
|----------|---------|
| Модалка 2FA на входе без поля кода | Отдельная разметка `#twoFactorModal` + inline critical CSS для OTP |
| Карточки безопасности не видны на проде | Переименование классов, scoped CSS, проверка в `deploy-cloud-app.sh` |
| Webhook ЮKassa иногда опаздывает | `POST /api/cloud/billing/sync-pending` после возврата с оплаты |
| Общий backend со студией | В репозитории вынесены только модули Cloud; auth монтируется в `main.py` |

## Метрики (ориентиры)

- Домен в продакшене: **cloud.lbl3d.info**
- Бесплатная квота по умолчанию: **5 ГБ** (`LBL_CLOUD_FREE_MB`)
- Максимальный размер файла: настраивается env (типично до **1 ГБ** на nginx)
